# Шахматы онлайн (веб‑страница, БЭМ)

Проект: одностраничное веб‑приложение “Шахматы онлайн” с комнатами (имитация “онлайна” между вкладками через `BroadcastChannel`), чатом и игрой на шахматной доске.

## Запуск (для проверки преподавателем)

Требования: Node.js (в окружении проекта корректно работает с Vite `6.4.x`).

```bash
cd shah-chess-online
npm i
npm run dev
```

Откройте ссылку из терминала.  
Чтобы проверить “онлайн”: откройте **вторую вкладку/окно** и войдите в **ту же комнату**.

Сборка:

```bash
npm run build
npm run preview
```

## Функциональность

- **Игровое поле**: клик по фигуре → подсветка допустимых ходов → клик по клетке.
- **Правила** (упрощённые, но “честные”):
  - легальные ходы с проверкой “нельзя оставлять короля под шахом”;
  - **шах**;
  - **рокировка**;
  - **взятие на проходе**;
  - **превращение пешки** (по умолчанию в ферзя).
- **Комнаты**: создание / вход по ID, ссылка комнаты копируется кнопкой.
- **“Онлайн”**: синхронизация ходов/состояния между вкладками через `BroadcastChannel`.
- **Протокол ходов** и **чат‑лог**.
- **Таймеры** (5 минут на игрока, базовая синхронизация по состоянию комнаты).

## БЭМ (что и где)

Вёрстка и стили выполнены по БЭМ (блоки/элементы/модификаторы). Примеры:

- **Блоки**: `page`, `header`, `layout`, `panel`, `form`, `button`, `input`, `chat`, `game`, `clocks`, `clock`, `board`, `moves`, `notice`, `footer`
- **Элементы**: `header__title`, `panel__section`, `board__square`, `board__piece`, `chat__list`, `moves__row`
- **Модификаторы**: `button--primary`, `panel--lobby`, `board__square--dark`, `board__square--selected`, `board__square--legal`

Файл стилей: `src/style.css`.

## Применённые принципы ООП

- **Инкапсуляция**: состояние позиции скрыто внутри класса `Position` и меняется только через методы (`src/chess/position.ts`).
- **Абстракция**: “транспорт комнаты” представлен интерфейсом `RoomTransport`, а конкретная реализация вынесена в `BroadcastChannelTransport` (`src/net/roomTransport.ts`).
- **Полиморфизм (через интерфейсы)**: код приложения работает с `RoomTransport`, не завися от конкретного канала обмена.
- **Композиция**: `ChessApp` использует `StandardRules`, `BoardView`, `RoomTransport`, не наследуя их (`src/ui/chessApp.ts`).

## Применённые принципы SOLID (с привязкой к коду)

- **S (Single Responsibility)**:
  - `Position` — хранение/сериализация состояния партии (FEN‑подобная строка) (`src/chess/position.ts`);
  - `StandardRules` — генерация/проверка ходов и применение хода (`src/chess/rules/standardRules.ts`);
  - `BoardView` — только отрисовка/клики по клеткам (`src/ui/components/boardView.ts`);
  - `ChessApp` — склейка UI + доменной логики + сети (`src/ui/chessApp.ts`).
- **O (Open/Closed)**:
  - можно добавить другой транспорт (например, WebSocket) как новую реализацию `RoomTransport` без переписывания `ChessApp` (`src/net/roomTransport.ts`, `src/ui/chessApp.ts`);
  - правила можно расширять новой реализацией (например, другой класс правил) без изменения `BoardView`.
- **L (Liskov Substitution)**:
  - любая реализация `RoomTransport` должна быть взаимозаменяемой для `ChessApp` (контракт: `send/onMessage/close`).
- **I (Interface Segregation)**:
  - интерфейс `RoomTransport` минимальный: нет лишних методов “на всякий случай” (`src/net/roomTransport.ts`).
- **D (Dependency Inversion)**:
  - `ChessApp` зависит от абстракции `RoomTransport`, а не от `BroadcastChannel` напрямую (`src/ui/chessApp.ts` → `src/net/roomTransport.ts`).

## Структура проекта

- `src/chess/*` — доменная модель шахмат (позиция, фигуры, ходы, правила)
- `src/net/*` — транспорт/сообщения комнаты
- `src/ui/*` — UI и контроллер приложения
